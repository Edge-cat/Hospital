# 问题修复 — 中文乱码 + 患者端路由 403

> 日期：2026-06-29

## 问题 1：预置数据大量 `?`

**原因**：Windows 下用 PowerShell 管道 `Get-Content | docker exec mysql` 导入 `hospital_his_seed_demo.sql` 时，UTF-8 中文被转成字面量 `0x3F`（`??`）。

**修复**：
1. 新增 `execution/scripts/import-sql-utf8.ps1`（`docker cp` + 容器内 `source`，强制 `utf8mb4`）
2. 新增 `execution/database/hospital_his_seed_fixup.sql`（清理乱码增量后重导）
3. 已对运行中 MySQL 执行 fixup + seed 重导

```powershell
powershell -ExecutionPolicy Bypass -File execution/scripts/import-sql-utf8.ps1 `
  -SqlFile execution/database/hospital_his_seed_fixup.sql
powershell -ExecutionPolicy Bypass -File execution/scripts/import-sql-utf8.ps1 `
  -SqlFile execution/database/hospital_his_seed_demo.sql
```

**注意**：勿再用 `Get-Content ... | docker exec -i mysql` 导入含中文的 SQL。

## 问题 2：用户端切换路由报「无管理端访问权限」

**原因**：患者登录后，用户 Web / 小程序仍请求：
- `GET /api/statistics/overview` → staff 专属（403 无权限访问该资源）
- `GET /api/admin/department/list` → admin 专属（403 无管理端访问权限）

首页 `Promise.all` 并发请求，切换路由时多页重复拉科室，出现多条红色 Toast。

**修复**：
| 层 | 变更 |
|----|------|
| 后端 | `CommonController` 新增 `GET /api/common/departments`、`GET /api/common/home-overview`（患者可读） |
| 用户 Web | `HospitalService.ts`、`api/index.js` 改调 common 接口 |
| 小程序 | `api/index.js`、`mock.js` 同步 |

**验证**：患者 token 访问 `/api/common/*` → 200；访问 `/api/admin/department/list` → 仍 403（符合预期）。
