# Bug 追踪目录

本目录用于记录 **东软云医院 HIS 小程序端**（`hospital-mini`）的缺陷与修复进度。

## 目录结构

| 路径 | 说明 |
|------|------|
| `bug.pdf` | 原始缺陷截图/描述（用户提交） |
| `fix-log/` | 结构化修复日志（Markdown），供后续会话续接 |

## 使用方式

1. 发现新 bug → 在 `fix-log/` 新增 `BUG-xxx-简述.md`，并更新 `fix-log/00-总览.md`
2. 开始修复 → 将对应条目状态改为「修复中」，填写方案与改动文件
3. 验证通过 → 状态改为「已修复」，补充验证步骤与 commit/日期
4. 每次 Cursor 会话开头 → 阅读 `fix-log/00-总览.md` 了解当前进度

## 关联模块

- 小程序源码：`hospital-mini/`
- 后端 API：`hospital-backend/`
- 开发审计：`log/03-小程序端.md`
