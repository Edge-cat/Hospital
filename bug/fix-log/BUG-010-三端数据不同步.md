# BUG-010 — 小程序提交后管理端无记录 / 余号不减少

| 字段 | 内容 |
|------|------|
| **状态** | 🟢 已修复 |
| **优先级** | P0 |
| **来源** | 用户反馈（三端已启动，数据不通） |
| **模块** | 挂号/预约全链路 |

---

## 现象

- 小程序在「预约挂号」页选 **今日** 提交后，医生卡片「余号 N」、时段「余 N」**不减少**
- 管理端（`hospital-admin`）**看不到**对应新记录
- 用户 Web 端（`hospital-user`）号源同样不刷新

---

## 根因

| # | 层级 | 问题 |
|---|------|------|
| 1 | 业务语义 | 选 **今日** = **当日挂号**（`POST /register`），非预约；管理端应看 **挂号管理**，非预约管理 |
| 2 | 前端提交 | 挂号分支未传 `doctorId`、`timeSlot` |
| 3 | 后端入库 | `RegisterServiceImpl` 未补全 `doctorId`/`patientId`，未存 `timeSlot` |
| 4 | 前端 UI | 医生卡片「余号」**固定统计今日**，选明天预约时不减少 |
| 5 | 用户 Web | 预约提交未传 `doctorId`，成功后未刷新 schedule |
| 6 | 数据库 | 已有库缺少 `register_order.time_slot` 列 |

---

## 修复方案

- [x] `RegisterOrder` + `schema.sql` + `alter_register_time_slot.sql`：新增 `time_slot`
- [x] `OrderContextHelper`：统一解析 `patientId` / `doctorId`
- [x] `RegisterServiceImpl`：入库 `timeSlot`、补全 ID、关联缴费
- [x] `AppointmentServiceImpl.buildSchedule`：合并当日挂号占用
- [x] `visit.vue`：挂号提交带 `doctorId`+`timeSlot`，成功后 `reloadAvailability()`
- [x] `hospital-user/appointment/index.vue`：提交带 `doctorId`，成功后 `loadSchedule()`
- [x] `deptIcon.js` / `visit.vue`：医生「余号」按**当前选中日期**汇总，切换日期/提交后同步刷新

## 部署注意

**已有 MySQL 库须执行一次：**

```sql
-- execution/database/alter_register_time_slot.sql
ALTER TABLE register_order
  ADD COLUMN time_slot VARCHAR(32) NULL COMMENT '挂号时段' AFTER register_time;
```

然后 **重启 IDEA 后端** 使 Java 改动生效。
