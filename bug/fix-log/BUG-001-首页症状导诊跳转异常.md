# BUG-001 — 首页症状标签 / 智能导诊跳转异常

| 字段 | 内容 |
|------|------|
| **状态** | 🟢 已修复 |
| **优先级** | P1 |
| **来源** | `bug.pdf` 第 1 页 |
| **模块** | 首页 `pages/index/index` |

---

## 现象

点击首页 **智能导诊** 入口，或搜索框内 **常见症状标签**（如「发热咳嗽」），进入错误/空白页面，而非预期科室详情。

---

## 复现步骤

1. 打开小程序首页  
2. 点击「智能导诊」或任一症状标签  
3. 观察跳转结果 — 可能为空白详情页或「暂无科室信息」列表页  

---

## 根因分析

| # | 根因 | 相关代码 |
|---|------|----------|
| 1 | 跳转 URL 中科室名 **未 URL 编码**，微信端中文 query 解析失败 | `index.vue` L216、L229 |
| 2 | 科室 API 加载失败时 `departments=[]`，症状匹配失败 fallback 到空科室列表 | 关联 [BUG-002](./BUG-002-预约挂号无科室.md) |
| 3 | 症状 `deptId` 硬编码，与后端自增 ID 可能不一致 | `index.vue` L142-148 `symptomHints` |
| 4 | 「智能导诊」实际跳转到科室列表，无独立导诊页（产品预期偏差） | `coreServices` L128 |

**关键代码**：

```211:219:hospital-mini/src/pages/index/index.vue
function goSymptom(item) {
  ...
  if (dept) {
    uni.navigateTo({ url: `/pages/department/detail?id=${dept.id}&name=${dept.name}` })
  } else {
    uni.navigateTo({ url: '/pages/department/department' })
  }
}
```

对比 `department.vue` 已正确使用 `encodeURIComponent(dept.name)`。

---

## 修复方案

### 方案 A — 最小修复（推荐先做）

- [x] `goSymptom` / `goDept` 中 `name` 使用 `encodeURIComponent(dept.name)`
- [x] 匹配失败时 `uni.showToast({ title: '未找到对应科室' })`，避免静默跳转空页
- [x] 先修复 [BUG-002](./BUG-002-预约挂号无科室.md) 环境/API，确保 `departments` 有数据
- [x] 首页「就诊回顾」「移动缴费」跳转前 `checkLogin()`（关联 BUG-004）

### 方案 B — 增强（可选）

- [ ] 症状匹配优先用 `department` 名称字段，弱化硬编码 `deptId`
- [ ] 智能导诊入口增加 loading / error 态（复用 `useDepartments`）
- [ ] 新增 triage 页或 `department?symptom=` 预选高亮（需产品确认）
