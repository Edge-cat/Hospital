## 现象

进入「预约挂号」页后，左侧 **科室列表为空**，无法选择科室与医生。

---

## 复现步骤

1. 首页 → 预约挂号（或 `/pages/visit/visit`）  
2. 观察 `DoctorSelect` 左侧科室栏 — 无任何选项  

---

## 根因分析

| # | 根因 | 说明 |
|---|------|------|
| 1 | **主因**：`VITE_API_BASE=/api` 在 `mp-weixin` 环境无效 | Vite proxy 仅 H5 开发有效；小程序请求 `/api/common/departments` 失败 |
| 2 | `visit.vue` 不展示 `useDepartments` 的 loading/error | 用户只见空列表，无「加载失败」提示 |
| 3 | 次要：`mapDeptList` 严格 `parentId === 0` | 若 DB 返回 `null` 会滤掉全部科室 |

**数据流**：

```
visit.vue onLoad
  → loadDepartments() [useDepartments.js]
    → GET /common/departments
      → 失败 → departments = []
        → DoctorSelect 空列表
```

**当前环境配置**（`hospital-mini/.env.development`）：

```env
VITE_USE_MOCK=false
VITE_API_BASE=/api
```

---

## 修复方案

### 1. 环境配置（必做）

**开发（微信开发者工具）** — 任选其一：

```env
# 方案 A：直连本地后端（需勾选「不校验合法域名」）
VITE_API_BASE=http://127.0.0.1:8080/api

# 方案 B：启用 Mock
VITE_USE_MOCK=true
```

**生产**：`.env.production` 改为真实 HTTPS 域名，并在微信公众平台配置 request 合法域名。

### 2. 代码改进

- [x] `visit.vue` 引入 `loading` / `error` / 重试按钮（参考 `department.vue`）
- [x] `deptIcon.js` `mapDeptList` 放宽：`(!d.parentId || d.parentId === 0)`
- [x] `.env.development` → `VITE_API_BASE=http://127.0.0.1:8080/api`
