# BUG-009 — 页面栈溢出 navigateTo webview count limit exceed

| 字段 | 内容 |
|------|------|
| **状态** | 🟢 已修复 |
| **优先级** | P0 |
| **来源** | 微信开发者工具控制台 |
| **模块** | 全局导航 |

---

## 现象

控制台反复报错：

```
navigateTo:fail webview count limit exceed
```

小程序页面栈达到微信上限（10 层）后，后续跳转全部失败。

---

## 根因

| # | 原因 |
|---|------|
| 1 | 各处直接使用 `uni.navigateTo`，栈深无控制 |
| 2 | `records.vue` 的 `onShow` 在未登录时**每次显示都** `navigateTo` 登录页，从登录页返回后再触发，无限叠栈 |
| 3 | Tab 页（预约/缴费）应用 `redirectTo`，子页仍用 `navigateTo` 层层叠加 |

---

## 修复方案

- [x] `utils/nav.js`：`safeNavigateTo`（栈 ≥7 降级 redirectTo）、`navigateToLogin`（防重复登录页）
- [x] `records.vue`：未登录显示「去登录」按钮，不再自动 navigateTo
- [x] 首页 / 我的 / 科室 / 公告 / VisitStatusBar 等改用 `safeNavigateTo`
- [x] Tab 跳转统一 `redirectTo`（TabBar、visit 详情入口等）

---

## 验证标准

- [ ] 连续点击首页入口、科室、详情、预约 10+ 次不再报 webview limit
- [ ] 未登录进入就诊回顾 → 显示提示，点「去登录」可正常跳转
- [ ] 登录返回后就诊回顾自动刷新
