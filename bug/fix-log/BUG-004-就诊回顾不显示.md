## 现象

从首页点击 **就诊回顾**，页面始终显示「暂无就诊记录」或空白，即使已登录且后端有数据。

---

## 复现步骤

1. **未登录**状态下，首页 → 就诊回顾  
2. 跳转登录页 → 登录成功 `navigateBack`  
3. 返回 records 页 — **仍无数据**  

或：已登录但 API 失败时，同样显示空态且无错误提示。

---

## 根因分析

| # | 根因 | 相关代码 |
|---|------|----------|
| 1 | **高概率**：仅用 `onMounted` 加载，从登录页返回 **不会再次触发** | `records.vue` L24-28 |
| 2 | 未登录时 `onMounted` 内 `navigateToLogin()` 后 `return`，永不加载 | 同上 |
| 3 | 首页「就诊回顾」入口 **不校验登录**（`mine.vue` 会先 checkLogin） | `index.vue` L222-225 |
| 4 | 无 `try/catch`，API 失败时 `list` 保持 `[]` | `records.vue` |
| 5 | 连锁依赖 BUG-003 登录失败 | `/record/list` 需 JWT |

**关键代码**：

```24:28:hospital-mini/src/pages/records/records.vue
onMounted(async () => {
  if (!checkLogin()) { navigateToLogin(); return }
  const res = await mpApi.records()
  list.value = res.data.list
})
```

后端 `GET /record/list` 需 Bearer token（`AuthInterceptor`）。

---

## 修复方案

### 方案 A — 最小修复（推荐）

- [x] 将加载逻辑改为 `onShow`（或 `onShow` + 防抖），登录返回后自动 reload  
- [x] 使用 `res.data?.list ?? []` 防御空数据  
- [x] 增加 `loading` 态与 `try/catch` 错误 toast  
- [x] 首页 `goPage('/pages/records/records')` 前 `checkLogin()`（`index.vue`）
