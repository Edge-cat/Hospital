# BUG-003 — 登录按钮无反应 / 微信登录失败

| 字段 | 内容 |
|------|------|
| **状态** | 🟡 部分修复（续跟 [BUG-005](./BUG-005-登录按钮仍无效.md)） |
| **优先级** | P0（阻塞鉴权） |
| **来源** | `bug.pdf` 第 3 页 |
| **模块** | 登录 `pages/login/login` |

---

## 现象

1. 点击 **登录** 按钮 seemingly 无反应（或仅短暂 loading 后 toast 消失）  
2. 点击 **微信登录** 显示「微信登录失败」  

---

## 复现步骤

1. 进入登录页 `/pages/login/login`  
2. 输入演示账号 `13800138000` / `123456`，点登录  
3. 或点击「微信一键登录」  

---

## 根因分析

| # | 根因 | 相关代码 |
|---|------|----------|
| 1 | API 不可达（同 BUG-002） | `request.js` → toast「网络连接失败」 |
| 2 | `handleLogin` 无 `catch`，错误仅依赖全局 toast | `login.vue` L35-49 |
| 3 | Mock 仅有 `/auth/login`，**无 `/auth/wx-login`** | `mock.js` L188+ |
| 4 | Mock 模式下 wx-login 返回空 `data`，token 为 undefined | `mock.js` 默认 resolve |
| 5 | 生产 `wx.demo=false` 且未配置 AppID/Secret | `application-prod.yml` |
| 6 | H5 环境 `uni.login({ provider: 'weixin' })` 必然失败 | 非微信容器 |

**演示账号**（后端 seed）：手机号 `13800138000`，密码 `123456`。

**后端 wx demo 模式**（`application.yml`）：code 为 `demo` 时可绑定 `13800138000`。

---

## 修复方案

### 1. 环境（与 BUG-002 同步）

```env
VITE_API_BASE=http://127.0.0.1:8080/api
# 或 VITE_USE_MOCK=true
```

后端需运行且 MySQL 有 patient/user 数据。

### 2. 代码改进

- [x] `mock.js` 补充 `POST /auth/wx-login` 分支，返回与 login 相同结构的 token/user  
- [x] `handleLogin` / `wxLogin` 增加 `catch`，展示 `e.message` 或后端 `message`  
- [x] `request.js`：HTTP 非 200 时尝试解析 body 中的 `message`（401 不再笼统「网络异常」）  
- [ ] 登录成功后可选：调用 `/patient/info` 补全 `phone/cardNo` 到 storage（暂缓）

### 3. 部署配置

- 开发：保持 `hospital.wx.demo=true`  
- 生产：配置 `HOSPITAL_WX_APP_ID` / `HOSPITAL_WX_APP_SECRET`
