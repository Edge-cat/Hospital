---

## 现象

多个页面 **缺少明显的「返回上一级」和「返回首页」入口**，用户在非 Tab 页（登录、科室详情、就诊记录等）容易迷失，无法快速回到首页。

典型场景：

- 登录页使用 `navigationStyle: custom`，**无系统返回键**
- 子页面仅依赖微信原生导航栏返回，**无「回首页」快捷方式**
- API 失败停留在子页时，TabBar 不可见（仅 index/visit/payment/mine 有自定义 TabBar）

---

## 影响页面

| 页面 | 当前导航 | 问题 |
|------|----------|------|
| `pages/login/login` | custom 导航 | 无返回/首页 |
| `pages/department/*` | 系统栏 | 无回首页 |
| `pages/notice/*` | 系统栏 | 无回首页 |
| `pages/records/records` | 系统栏 | 无 TabBar、无回首页 |
| `pages/mine/my-*` | 系统栏 | 无 TabBar |
| `pages/payment/result` | 仅有「返回首页」文案链 | 风格不统一 |

**有 TabBar 的页面**：`index`、`visit`、`payment`、`mine`（相对正常）

---
