# BUG-007 — 科室描述缺失 & 预约挂号号源显示错误

| 字段 | 内容 |
|------|------|
| **状态** | 🟢 已修复 |
| **优先级** | P1 |
| **来源** | 用户截图（2026-06-29） |
| **模块** | 科室介绍 / 预约挂号 |

---

## 现象

| 截图 | 问题 |
|------|------|
| 图1 科室列表 | 全部显示通用文案「提供门诊与住院诊疗服务」，负责人为「—」 |
| 图2 科室详情 | 简介「暂无简介」，负责人空白 |
| 图3 预约挂号 | 医生卡片一律显示红色「号源已满」，但下方时段仍有「余4」等 |

---

## 根因

| # | 根因 |
|---|------|
| 1 | 后端 `SysDepartment` 返回 `description`，前端读 `desc` / `leader`，字段未映射 |
| 2 | 后端无 `leader` 字段，需客户端 enrich |
| 3 | `/doctor/list` 不返回 `remaining`，`undefined > 0` 为 false → 误判「号源已满」 |

---

## 修复方案

- [x] `deptIcon.js`：`DEPT_META` + `description`→`desc`，补全 leader / commonDiseases
- [x] `visit.vue`：加载医生后调用 `/appointment/schedule` 汇总今日 `remaining`
- [x] `DoctorSelect.vue`：仅 `remaining != null` 时展示号源徽章

## 验证标准

- [ ] 科室列表：各科有独立简介与负责人（如内科「张主任」）
- [ ] 科室详情：简介非「暂无简介」，常见疾病标签可见
- [ ] 预约挂号：有号源医生显示「余号 N」，仅真正满号显示「号源已满」
