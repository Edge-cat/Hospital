## 现象

在微信开发者工具中，登录页已预填 `13800138000` / `123456`，点击蓝色 **「登录」** 按钮仍无成功跳转；用户感知为「点击无效」。

> BUG-003 曾标记为已修复，但 **人工复测未通过**，本条目作为回归/续跟跟踪。

---

## 复现步骤

1. `npm run dev:mp-weixin`，开发者工具导入 `dist/dev/mp-weixin`
2. 未启动后端或已启动后端均可复现不同分支
3. 进入 `pages/login/login`，点击「登录」

---

## 根因分析

| # | 根因 | 说明 |
|---|------|------|
| 1 | **后端未启动 / API 失败** | toast 一闪而过，用户以为按钮无反应 |
| 2 | **编译产物未更新** | 修改 `.env` 或源码后未重启 dev 编译，仍跑旧包 |
| 3 | **登录成功但 `navigateBack` 无栈** | 直接进入登录页时返回失败，未 fallback 首页 |
| 4 | **`navigationStyle: custom`** | 无系统导航栏，失败时用户无法离开登录页 |
| 5 | **`api.js` 动态 import** | mp-weixin 不支持 `import().then`，报 `./request.js.then is not a function` |

---

## 修复方案

- [x] `api.js` 改为静态 `import { request }`，移除动态 `import().then`（**根因**）
- [x] `navigateAfterLogin` 改为 `goBackOrHome()`（有栈返回，无栈 `reLaunch` 首页）
- [x] 登录页增加 `PageNav`（返回 / 首页）
- [x] 增加页面内 `errorMsg` 持久展示 API 错误
- [ ] 复测：后端 `:8080` 启动 + 「不校验合法域名」
- [ ] 复测：Mock 模式 `VITE_USE_MOCK=true` 离线登录
