# BUG-008 — 预约提交后余号 / 时段不刷新

| 字段 | 内容 |
|------|------|
| **状态** | 🟢 已修复 |
| **优先级** | P1 |
| **来源** | 用户反馈（预约成功后号源不变） |
| **模块** | 预约挂号 `pages/visit/visit` |

---

## 现象

提交预约成功后，医生卡片「余号 N」与各时段「余 N」**不减少**，与真实预约量脱节。

---

## 根因（数据连通断裂）

| # | 层级 | 问题 |
|---|------|------|
| 1 | 前端提交 | `POST /appointment` **未传 `doctorId`** |
| 2 | 后端入库 | 无 `doctorId` 时 `/appointment/schedule` 统计不到该预约 |
| 3 | 前端 UI | 提交成功后**未重新拉取** schedule / 医生余号 |
| 4 | Mock | `buildDoctorSchedule` 用固定公式，**未读取** `myAppointments` |

---

## 修复方案

- [x] `visit.vue`：提交时带 `doctorId`；成功后 `reloadAvailability()`
- [x] `visit.vue`：关闭成功弹窗时再次刷新（继续预约场景）
- [x] `AppointmentServiceImpl.create`：按 doctorName+department 补全 `doctorId`
- [x] `mock.js`：排班按 `myAppointments` 扣减配额（每时段 quota=4）
