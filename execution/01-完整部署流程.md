## 三、数据库部署（第三方可复制执行）

### 方式 A：一键 SQL（推荐）

```bash
mysql -u root -p < execution/database/hospital_his_full.sql
```

脚本包含：
1. `CREATE DATABASE hospital_his`
2. 全部 27 张表结构
3. 演示种子（**充盈版**：多患者/多业务/多状态，各页面有内容）

已有库仅补数据（**须用 UTF-8 安全脚本，勿用 PowerShell 管道**）：

```powershell
powershell -ExecutionPolicy Bypass -File execution/scripts/import-sql-utf8.ps1 `
  -SqlFile execution/database/hospital_his_seed_demo.sql
```

### 方式 B：Docker MySQL

```bash
cd execution
docker compose up -d mysql
# 等待 healthy 后，数据已通过 docker-entrypoint-initdb.d 自动导入
```

### 方式 C：手动

```sql
CREATE DATABASE hospital_his DEFAULT CHARSET utf8mb4;
USE hospital_his;
-- 粘贴 execution/database/hospital_his_full.sql 中 USE 之后的内容
```

### 生产注意

- 修改默认账号密码（`sys_user` 表）
- 勿在生产环境重复执行全量脚本（会重复 INSERT）
- 后端生产 profile 设置 `spring.sql.init.mode=never`（已在 `application-prod.yml`）

## 四、后端部署

### 1. 环境变量

| 变量 | 说明 | 示例 |
|------|------|------|
| `HOSPITAL_DB_URL` | JDBC URL | `jdbc:mysql://127.0.0.1:3306/hospital_his?...` |
| `HOSPITAL_DB_USER` | 数据库用户 | `hospital` |
| `HOSPITAL_DB_PASSWORD` | 数据库密码 | `***` |
| `HOSPITAL_JWT_SECRET` | JWT 密钥（≥256 bit） | 随机长字符串 |
| `HOSPITAL_WX_DEMO` | 微信演示模式 | `false` |
| `HOSPITAL_WX_APP_ID` | 小程序 AppID | `wx***` |
| `HOSPITAL_WX_APP_SECRET` | 小程序 Secret | `***` |
| `HOSPITAL_CORS_ORIGINS` | 允许的前端域名 | `https://admin.xxx.com,https://user.xxx.com` |

### 2. 打包与启动

```bash
cd hospital-backend
mvn -q clean package -DskipTests
java -jar target/hospital-backend-1.0.0-SNAPSHOT.jar --spring.profiles.active=prod
```

### 3. 验证

```bash
curl http://127.0.0.1:8080/api/common/meta
cd hospital-backend/scripts && powershell -File smoke-test.ps1
```

## 五、前端部署

### 1. 构建

```powershell
powershell -ExecutionPolicy Bypass -File execution/scripts/build-frontends.ps1
```

产物：
- `hospital-admin/dist`（生产 base: `/admin/`）
- `hospital-user/dist`（生产 base: `/user/`）
- `hospital-mini/dist/build/h5`（H5）或 `dist/build/mp-weixin`（小程序）

### 2. Nginx 部署（同源反代 `/api`）

参考 `execution/nginx/hospital.conf`：

| 路径 | 指向 |
|------|------|
| `/admin/` | 管理端 dist |
| `/user/` | 用户端 dist |
| `/api/` | `http://127.0.0.1:8080` |

`.env.production` 中 `VITE_API_BASE=/api` 即可（Nginx 同源）。

### 3. 小程序

1. 修改 `hospital-mini/.env.production` → `VITE_API_BASE=https://api.your-domain.com/api`
2. 修改 `manifest.json` 中真实 AppID
3. 微信公众平台配置 request 合法域名
4. `npm run build:mp-weixin` 后上传体验版
